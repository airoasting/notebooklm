# 리뷰 + 수정 사이클

이 스킬은 내부 리뷰(1~6단계)와 외부 리뷰(7~9단계)를 순서대로 수행한다. 각 단계가 끝나면 산출물에 상태 마커를 달고 다음 단계로 자동 진행한다.

## 시작 전 준비

`user-book-toc.md`를 읽고 목차와 페르소나를 파악한다.

## 활성 폴더 결정

`draft/` 하위 폴더 중 사전순 가장 최신 폴더를 활성으로 선택한다.

```bash
ACTIVE=$(ls -1d draft/*/ 2>/dev/null | sort | tail -n 1)
ACTIVE="${ACTIVE%/}"
[ -z "${ACTIVE}" ] && { echo "활성 폴더 없음. /research를 먼저 실행하세요."; exit 1; }
echo "활성 프로젝트 폴더: ${ACTIVE}"
```

이후 모든 경로는 `{ACTIVE}`로 표기한다.

# 공통 규칙

## 심각도 분류 (모든 리뷰어 공통)

모든 지적 사항은 다음 3단계 중 하나로 표기한다.

| 심각도 | 의미 | 반영 의무 |
|--------|------|-----------|
| 🔴 필수 | 반드시 수정해야 출판 가능. 사실 오류, 논리 모순, 목차 누락, 작성 규칙 위반, 맞춤법 오류, 비문 | 100% 반영 |
| 🟡 권장 | 수정하면 품질이 올라감. 표현 개선, 흐름 보완, 예시 보강, 어색한 표현 | 판단 후 반영 |
| 🟢 참고 | 고치지 않아도 됨. 스타일 선호, 대안 제안 | 무시 가능 |

## 10점 만점 점수 anchor (모든 리뷰어 공통)

점수는 객관적으로 산출 가능해야 한다. 모든 리뷰어는 다음 anchor를 따른다.

| 점수 | 결함 상태 | 의미 |
|------|-----------|------|
| 10점 | 🔴 0건, 🟡 0건, 🟢 0~2건 | 출판 즉시 가능 |
| 9점 | 🔴 0건, 🟡 1~2건 | 합평 통과 가능 (게이트 라인) |
| 8점 | 🔴 0건, 🟡 3~5건 | 한 번 더 수정 필요 |
| 7점 | 🔴 1건 또는 🟡 6~10건 | 부분 재작업 필요 |
| 6점 | 🔴 2건 또는 🟡 11~15건 | 상당한 수정 필요 |
| 5점 이하 | 🔴 3건 이상 또는 🟡 16건 이상 | 재작성 수준 |

🔴 1건이라도 있으면 점수 상한선은 7점이다. 점수와 무관하게 🔴 0건이 되어야 통과다.

## 지적 사항 출력 형식 (모든 리뷰어 공통)

각 지적은 아래 5개 필드를 포함한다.

```
[심각도] [위치] - [제목]
- 원문: "..." (있으면 정확히 인용, 길면 첫 30자 + ...)
- 문제점: 구체적으로 무엇이 잘못되었는가
- 수정 제안: 어떻게 고치면 좋은가
```

리뷰 파일 마지막에는 다음 요약 블록을 붙인다.

```
## 점수
- 🔴 필수: N건
- 🟡 권장: N건
- 🟢 참고: N건
- 종합 점수: X / 10
- 통과 여부: 통과 / 재수정
```

# 내부 리뷰 (1~6단계)

## 1단계: 비평가 리뷰 (Critic)

### 역할

냉정한 비평가로서 초안을 평가한다. 칭찬보다 개선점을 먼저 말한다. 목소리는 차갑지만, 지적은 구체적으로 한다.

### 입력 / 출력

- 입력: `{ACTIVE}/03_draft-v1.md`
- 출력: `{ACTIVE}/04_review-red.md`

### 평가 항목 + 심각도 규칙

| 항목 | 🔴 필수 기준 | 🟡 권장 기준 |
|------|--------------|---------------|
| 근거 없는 주장 | 데이터·출처 없이 단정한 주장 1건 | 약한 근거를 둔 주장 |
| 논리 비약 | 인과 사이가 끊어진 결론 | 흐름이 매끄럽지 않은 전개 |
| 중복 내용 | 같은 내용이 두 챕터 이상 반복 | 인접 단락 표현 중복 |
| 작성 규칙 위반 | em dash 사용, 어미 혼용, `**` 잔존, 페르소나 일탈 | 부호 비통일, 표현 보강 가능 |
| 목차 누락 | user-book-toc.md 명시 항목 누락 | 분량이 가이드 하한 미달 |

## 5단계: 합평 1라운드 (Ensemble — 5인)

### 합평 운영 규칙

5명의 평가자가 **동일 원고를 각자의 관점으로 읽고 독립적으로 채점**한다. 평가는 한 파일 안에 5개 섹션으로 나란히 적는다. 점수는 각자 10점 만점, 마지막에 평균 점수를 낸다.

평가자 5인:
1. **사실 검증가 (Fact Checker)**
2. **구조 검증가 (Architect)**
3. **논리 검증가 (Logician)**
4. **독자 검증가 (Reader Proxy)**
5. **서식 검증가 (Format Auditor)**

#### 독립 채점 보장 (Anchor Bias 방지)

LLM이 한 번에 5인 점수를 다 쓰면 첫 평가자 점수에 나머지가 끌려간다. 이를 막기 위해 다음 운영 규칙을 강제한다.

1. **순차 작성**: 평가자 1을 끝낼 때까지 평가자 2의 점수를 미리 쓰지 않는다. 한 평가자 섹션은 [지적 사항 → 점수 → 다음 평가자] 순으로만 진행한다
2. **컨텍스트 격리 권장**: 가능하면 평가자별로 별도 호출(서브에이전트 또는 별도 대화)에서 채점한 뒤 결과를 합친다
3. **점수 사전 공개 금지**: 같은 라운드 안에서 다른 평가자의 점수를 참조하지 않는다. "다른 평가자가 8점이라 나도 8점"은 금지
4. **anchor 강제**: 점수는 위 공통 anchor 표(🔴/🟡 건수 → 점수)로 **계산하여 산출**한다. "감으로 8점" 금지
5. **편향 검토 줄**: 각 평가자 섹션 마지막에 한 줄을 적는다 — `편향 검토: 다른 평가자 의견을 보지 않고 산출함`

### 입력 / 출력

- 입력: `{ACTIVE}/07_draft-v3.md`
- 출력: `{ACTIVE}/08_ensemble-review-1.md`

### 평가자별 루브릭

#### 5-1. 사실 검증가 (Fact Checker)

| 항목 | 🔴 필수 기준 | 🟡 권장 기준 |
|------|--------------|---------------|
| 사실 정확성 | 검증 가능한 사실 오류 1건 | 출처는 맞지만 표현이 모호 |
| 출처 누락 | 통계·인용에 출처 없음 | 출처는 있으나 형식 비일관 |
| 데이터 일관성 | 같은 수치가 챕터 간 불일치 | 단위 표기 비통일 |
| 기능 정보 정합성 | user-book-toc.md "기준 연도" 위반 또는 기능 설명 오류 | 기능 명칭 표기 비통일 |

#### 5-2. 구조 검증가 (Architect)

| 항목 | 🔴 필수 기준 | 🟡 권장 기준 |
|------|--------------|---------------|
| 목차 일치 | user-book-toc.md 목차 항목 누락 | 하위 섹션 1개 누락 |
| 분량 밸런스 | 한 부가 다른 부의 2배 이상 또는 1/2 이하 | 챕터 간 분량 편차가 큼 |
| 브릿지 연결 | "N부를 마치며" 누락 또는 다음 부와 연결 끊김 | 브릿지가 있으나 약함 |
| 캐릭터 등장 | 1부 캐릭터가 후반부에서 소실 | 등장 빈도 불균형 |

#### 5-3. 논리 검증가 (Logician)

| 항목 | 🔴 필수 기준 | 🟡 권장 기준 |
|------|--------------|---------------|
| 근거 없는 주장 | 데이터·출처 없이 단정 1건 | 근거가 약한 추론 |
| 논리 비약 | 전제와 결론 사이 단계 누락 | 연결어가 부적절 |
| 중복 | 같은 메시지가 활용법 간 반복 | 한 챕터 내 표현 중복 |
| 자기모순 | 앞 챕터와 뒤 챕터 주장이 충돌 | 미묘한 뉘앙스 차이 |

#### 5-4. 독자 검증가 (Reader Proxy)

| 항목 | 🔴 필수 기준 | 🟡 권장 기준 |
|------|--------------|---------------|
| 용어 미설명 | 전문 용어 미설명으로 진행 불가 | 설명 부족 |
| 따라 하기 가능 | 절차가 끊겨 실행 불가 | 한 단계 모호 |
| 흥미 유지 | 도입부에서 책을 덮고 싶음 | 중간 챕터 흥미 약화 |
| 페르소나 톤 | "옆자리 선배" 톤이 깨져 가르치는 어조 | 위트가 부족 |

#### 5-5. 서식 검증가 (Format Auditor)

| 항목 | 🔴 필수 기준 | 🟡 권장 기준 |
|------|--------------|---------------|
| 시각 자료 | 챕터·활용법에 [그림 N] 플레이스홀더 누락 | 캡션 형식 비통일 |
| 부호 사용 | em dash(—, –) 사용 1건 | 불릿 부호 비통일(• 외 사용) |
| 마크다운 잔존 | `**` 등 마크다운 기호 본문에 잔존 | 헤딩 레벨 비일관 |
| 표 포맷 | user-book-toc.md 퍼블리싱 사양 위반 | 표 헤더 표기 비통일 |
| 핵심 정리 박스 | 활용법에 박스 누락 | 박스 형식 비통일 |

### 통과 기준 (게이트)

- **🔴 0건** AND **5인 평균 9점 이상** → 통과
- 🔴가 1건이라도 있으면 점수와 무관하게 재수정
- 9점 미만이면 수정 후 합평 1회 추가 (최대 3라운드)
- 3라운드 후에도 미통과면 현재 버전으로 진행하되, 미통과 사유를 `{ACTIVE}/10_ensemble-review-2.md`에 명시

### 3라운드 발생 시 파일 규칙

번호 체계를 유지하기 위해 **같은 번호의 파일을 덮어쓴다**. 새 파일을 추가하지 않는다.
- 기존 `10_ensemble-review-2.md`를 덮어쓴다
- 수정본도 `09_draft-v4.md`를 덮어쓴다
- 덮어쓰기 전, 직전 버전을 `{ACTIVE}/_backup/` 폴더에 타임스탬프 붙여 보관한다 (예: `09_draft-v4_20260504_1530.md`)

### 합평 통과 후

# 외부 리뷰 (7~9단계)

## 7단계: 편집자 리뷰 (Editor)

### 역할

20년차 출판사 편집자다. "독자가 이 책을 끝까지 읽게 만들 수 있는가"가 가장 중요한 기준이다. 칭찬보다 개선점을 먼저 말한다.

### 입력 / 출력

- 입력: `{ACTIVE}/11_draft-final.md`
- 출력: `{ACTIVE}/12_review-editor.md`

### 평가 항목 + 심각도 규칙

| 항목 | 🔴 필수 기준 | 🟡 권장 기준 |
|------|--------------|---------------|
| 구성력 | 부와 부 사이 서사가 끊김 | 흐름이 무난하나 임팩트 약함 |
| 서사력 | 캐릭터 일관성 붕괴, 도입 훅 부재 | 캐릭터 묘사가 평면적 |
| 톤 일관성 | 페르소나가 챕터마다 다름 | 챕터별 톤 강도 편차 |
| 분량 밸런스 | 한 부가 책 분량의 30% 초과 또는 5% 미만 | 챕터 간 편차 |
| 독자 경험 | 독자가 중반에 책을 덮을 만한 지점 존재 | 중간 휴식 포인트 부족 |
| 시장 경쟁력 | 동일 주제 기존 도서와 차별점이 보이지 않음 | 차별점은 있으나 약함 |
| 완성도 | 부록·참고문헌 누락 | 부록 항목 비통일 |

(공통 형식)

- 종합 점수: X / 10
- 통과 여부: 통과 / 재수정

### 8-1. 마케터 리뷰 (Marketer) — 참고용

#### 역할

서점 MD에게 한 줄로 책을 소개해야 하는 마케터다. **본문 수정에는 반영하지 않는다.** 단, `/publish` 단계에서 메타데이터 생성에 사용된다.

#### 입력 / 출력

- 입력: `{ACTIVE}/11_draft-final.md`
- 출력: `{ACTIVE}/13_review-marketer.md`

#### 평가 항목 + 심각도 규칙

| 항목 | 🔴 필수 기준 | 🟡 권장 기준 |
|------|--------------|---------------|
| 타겟 명확성 | 누구를 위한 책인지 한 줄 정의 불가 | 타겟이 두 그룹 이상으로 흐림 |
| 제목 파워 | 제목만으로 핵심 가치 전달 불가 | 제목이 평이 |
| 프롤로그 흡인력 | 첫 3페이지에 후킹 부재 | 후킹은 있으나 약함 |
| 목차 매력도 | 목차만 보고 흥미가 안 생김 | 일부 챕터 제목이 평이 |
| 시의성 | 1년 안에 정보가 chao화될 위험 | 일부 사례가 단기 유통기한 |

#### 출력 템플릿

## 한 줄 소개 후보 3개
1. ...
2. ...
3. ...

## 지적 사항
(공통 형식)

## 점수
- 종합 점수: X / 10
